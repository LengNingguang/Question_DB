# 项目定义

## V1 要做什么
- 建立题库主库，存放试卷、题目、图片、答案、题目级聚合统计、难度占位信息。
- 建立标准化清洗包格式，避免人工直接改数据库。
- 提供稳定的只读查询 API。
- 提供内部版和半公开版导出。
- 通过环境变量把源码仓与服务器运行环境解耦。

## V1 不做什么
- 不保存考生个人原始成绩。
- 不做复杂后台管理页面。
- 不直接复用旧赛事运营数据库作为主库。
- 不在 V1 里实现 IRT 等需要逐题匿名作答数据的复杂模型。
- 不把生产 SQLite、生产原始资料或服务器密钥提交进源码仓。

## 源码仓职责
- 保存 schema、脚本、API、样例 bundle、测试和文档。
- 提供本地开发和验证所需的样例资源。
- 通过环境变量或命令行参数连接服务器上的生产数据库。

## 服务器职责
- 保存生产 SQLite、生产 assets、原始资料和运行日志。
- 运行 FastAPI 进程并对外提供只读查询接口。
- 承担备份、回滚和服务重启。

## 统一术语
- `paper`: 一套试卷。
- `question`: 一道题目。
- `asset`: 题图、答案图或附图。
- `stats`: 题目聚合统计。
- `difficulty`: 难度字段，包含人工标签与规则评分。
- `bundle`: 一套经过清洗、可直接导入的中间格式资料包。

## 权限边界
- 内部主库保存题干、答案、来源、聚合统计和导入日志。
- 半公开导出只允许暴露题目元数据与许可公开的聚合统计。
- 原始成绩表只作为统计脚本输入，不进入公开层。
- 所有服务器信息都使用占位符表达，真实值仅在部署环境中注入。
