# FAQ

## 为什么生产 SQLite 不进 Git？
因为生产数据库是运行态文件，包含持续变化的数据、日志关联和部署环境路径。把它放进源码仓会让版本管理混乱，也更容易误提交敏感信息。

## 源码仓如何连接服务器上的 SQLite？
通过环境变量或私有部署配置注入路径，例如 `QUESTION_BANK_DB_PATH`。仓库中只保留变量名和占位符，不保留真实服务器地址或目录。

## 为什么仍然用 SQLite？
当前题量和访问模式下，SQLite 足够稳定，部署也轻。后续若并发和写入需求明显提升，再迁移到 Postgres。

## GitHub Actions 能直接当生产 API 吗？
不能。Actions 适合测试、校验和构建，不适合长期对外提供动态 HTTP 服务。生产 API 应在你们自己的服务器上运行，并直接读取本机 SQLite。

## 半公开导出会包含答案吗？
默认不会。使用 `scripts/export_data.py --public` 导出的文件会自动去掉答案字段。
