# CPHOS 题目数据库 V1 实施规划

## Summary
- 目标是在 2 周内交付一个可持续维护的题库 V1：完成题目主库、原始资料整理规范、导入清洗流程、只读查询 API、基础统计、难度评定预留。
- V1 固定范围为：理论题与实验题、题干 LaTeX、题图、答案、题目来源、题目聚合统计。成绩仅保留题目级聚合数据，不保存个人成绩。
- 技术路线固定为：`GitHub 私有源码仓 + 服务器上的 SQLite 主库 + 服务器资产目录 + FastAPI 只读接口 + JSONL/CSV 导出`。
- 源码仓只保存 schema、脚本、API、样例与文档；生产 SQLite、原始资料、题图和运行日志全部放在自有服务器上，并通过环境变量注入路径。

## 进度规划
### 阶段 0：项目启动与定标
- 产出 `docs/project_definition.md`，统一术语、权限边界、V1 范围、非目标清单。
- 明确“源码仓”和“服务器运行环境”的职责边界。
- 完成标准：任何组员读完 10 分钟内都能复述项目边界和部署边界。

### 阶段 1：资料盘点与风险清单
- 使用 `scripts/register_raw_assets.py` 生成 `docs/material_inventory.csv`。
- 对每份原始资料补充状态、年份、可解析性、风险标签。
- 完成标准：每份资料都能定位到路径，有状态标记。

### 阶段 2：字段设计与样例定稿
- 固化表结构、枚举值、题图命名和样例清洗包。
- 维护 `docs/field_dictionary.md` 与 `samples/demo_bundle/`。
- 完成标准：新增录入不需要再临时发明字段。

### 阶段 3：仓库骨架与服务器配置骨架
- 初始化源码仓目录、SQLite schema、样例库生成脚本。
- 增加环境变量约定、`.env.example`、部署模板和部署文档。
- 明确开发样例库与生产库分离，生产库不进入 Git。
- 完成标准：新成员能一键初始化本地样例库，并看懂服务器所需配置项。

### 阶段 4：导入链路打通
- 使用 `scripts/validate_bundle.py` 和 `scripts/import_bundle.py` 实现清洗包导入。
- 通过 `import_runs` 留痕，支持 dry-run 和 commit。
- 完成标准：至少一套卷子能从清洗包入库。

### 阶段 5：服务器部署 API
- 实现 `/papers`、`/questions`、`/questions/{id}`、`/search`。
- API 通过环境变量读取服务器本机 SQLite，而不是读取仓库内生产库。
- 补充启动命令、进程守护、日志位置和服务重启说明。
- 完成标准：组员无需直接查 SQLite，运维同学能按文档部署服务。

### 阶段 6：成绩聚合与统计层
- 用 `scripts/import_stats.py` 从逐题得分 CSV 生成聚合统计。
- 统一统计版本与来源口径。
- 完成标准：能抽样核对每题均分、方差、满分率和零分率。

### 阶段 7：首批全量录入
- 逐套试卷导入并执行质量检查。
- 补齐缺失项清单与状态字段。
- 完成标准：核心历史题目集完成首轮入库。

### 阶段 8：难度评定预留
- 使用 `scripts/calculate_difficulty.py` 生成规则难度。
- 明确后续 IRT 需要匿名化逐题作答数据。
- 完成标准：有统计数据的题均有基础难度值或数据不足标记。

### 阶段 9：文档、验收与交接
- 完成录入规范、维护手册、FAQ、字段字典和部署文档。
- 交接时同时交付源码仓说明与服务器运维说明。
- 完成标准：未参与开发的成员能独立完成一次题目录入，并完成一次服务器部署演练。
